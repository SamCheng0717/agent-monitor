agent-monitor 使用说明

这是什么？

AI + 人工半监督的客服质量监控与提示词演进飞轮。每天凌晨 cron 自动跑： 从 Dify 拉所有对话 → 找违规 → 主管 LLM 优化 prompt → 三阶段测试 → 通过则生成 pending 候选 → 等你审批 → 推到 Dify 生产。 你只需要每天 5 分钟点几下网页。

📅 每日操作（5 分钟）

  1. 打开 192.168.103.66:8090
  2. 看顶部 4 张 KPI 卡：
    • 留资率带涨跌箭头：↑ 表示昨天比前天好
    • 待审候选显示数字 = advisor 凌晨生成了新 prompt 等你审
  3. 看「待审候选」区，如有：
    • 点 "查看 diff" → 弹窗显示新 prompt vs 当前的差异
    • 看顶部 4 个字段：改了哪个模块、违反什么规则、对留资影响、三阶段通过率
    • 合理就点 "批准并发布"（自动写本地 + 推 Dify 生产）
    • 不合理点 "驳回"（删 pending，明天 advisor 重新尝试）
  4. 「人工反馈」框里写下今天观察到的客服群问题（看到顾客抱怨、客服吐槽都可以记），下次 advisor 自动读
  5. 关浏览器

🔄 每周一次（10 分钟）

  1. 滚到「回归测试集挖掘」区
  2. 点 "扫历史 reports"，按违规规则分组列出真实候选对话
  3. 每组挑 1-2 条最典型的，勾上 ☑️
  4. 点 "晋升选中" → 写入真实回归集
  5. 意义：让稳定性测试越来越接近真实生产场景，不靠手写脑补

🚨 出问题时

症状怎么处理
新版本上线后 AI 答得很糟 版本时间线点上一个圆点 → "回滚 + 同步推 Dify"
怀疑本地 prompt 和 Dify 不一致 点最新版本圆点 → "回滚 + 同步推 Dify"（强制对齐）
改完合成回归集想立刻验证 "测试集状态"卡片右上角 → "立即跑回归"（30 秒）
想看某天为什么留资掉 折线图点该天的点 → 弹窗看完整劣质对话清单
advisor 连续几天 failed 看 "近 7 日 advisor 行动" 表，多半是某条规则总改不好。在反馈框里写人工建议
仪表盘打不开 SSH：pkill -f uvicorn && cd /www/wwwroot/agent-monitor && nohup venv/bin/uvicorn web.app:app --host 0.0.0.0 --port 8090 &

📖 名词解释

飞轮
每天循环：监控 → 发现违规 → 优化 → 测试 → 审批 → 上线 → 次日验证
pending
advisor 生成的候选 prompt，写在 prompts/pending/，等你点"批准"才上 Dify
optimize 集
从生产失败自动累积，驱动每天迭代。candidate 必须 100% 通过
holdout 集
cases.json 随机分出的 20%，防过拟合。candidate 要 ≥ 75% 通过
regression 集
人工维护的稳定测试，守稳定性底线。candidate 要 ≥ 95% 通过才能进 pending
三阶段闸门
candidate 必须依次过 optimize / holdout / regression，任一阶段失败就把失败 case 喂回主管 LLM 重试，最多 5 轮
违规
AI 回复违反 system_prompt.md 的某条具体规则。每条违规含 rule / evidence / impact 三段
留资
顾客留下了微信号或手机号 — 唯一核心 KPI

🔁 飞轮架构

[02:00 cron]   monitor.py     拉 Dify 全部对话 + 留资识别 + 违规检测
                              ↓
                              reports/<date>.json
                              ↓
[02:30 cron]   advisor.py     提取测试用例 → 主管 LLM 迭代 → 三阶段闸门
                              ↓
                              prompts/pending/pending_v00X_<date>.md
                              ↓
                              钉钉通知 "候选 v00X 待审核"
                              ↓
[人工 5 分钟]                   网页 diff 审核
                              ↓
                              批准 → 写本地 + 推 Dify publish
                              ↓
                              生产 chatflow 应用立即生效
                              ↓
[次日]                          monitor 验证修复 / 发现新失败
                              ↻ 循环

📂 关键文件位置（仅 SSH 用）

/www/wwwroot/agent-monitor/prompts/system_prompt.md — 当前生效
/www/wwwroot/agent-monitor/prompts/versions/ — 历史归档
/www/wwwroot/agent-monitor/prompts/pending/ — 待审候选
/www/wwwroot/agent-monitor/reports/<date>.json — 每日数据
/www/wwwroot/agent-monitor/tests/cases.json — 自动训练集
/www/wwwroot/agent-monitor/tests/regression_set*.json — 回归测试集
/www/wwwroot/agent-monitor/feedback/pending.md — 人工反馈
/www/wwwroot/agent-monitor/logs/ — cron 输出日志
